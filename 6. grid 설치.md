## 5. ASM 공유 디스크 확인
```bash
[root@psk1 ~]# oracleasm listdisks
[root@psk2 ~]# oracleasm listdisks
```
### 조회 안 될 시
  
  - psk1
```bash
# 파티션 생성 (psk1)
for disk in sdb sdc sdd sde sdf; do
	echo -e "n\np\n1\n\n\nw" | fdisk /dev/$disk

done
partprobe

# ASM 레이블링 (psk1에서만 수행)
oracleasm createdisk CRS1 /dev/sdb1
oracleasm createdisk CRS2 /dev/sdc1
oracleasm createdisk CRS3 /dev/sdd1
oracleasm createdisk DATA1 /dev/sde1
oracleasm createdisk FRA1 /dev/sdf1

# 확인
oracleasm listdisks
# CRS1 CRS2 CRS3 DATA1 FRA1 나오면 성공
```
   - psk2
```bash
# 파티션 생성 (psk2)
for disk in sdb sdc sdd sde sdf; do
	echo -e "n\np\n1\n\n\nw" | fdisk /dev/$disk

done
partprobe

# psk2는 레이블링(createdisk) 하지 않습니다.
# psk1에서 레이블링한 공유 디스크를 scandisks로 인식합니다.
oracleasm scandisks

# 확인
oracleasm listdisks
# CRS1 CRS2 CRS3 DATA1 FRA1 나오면 성공
```

```

FRA1

DATA1
FRA1
```

## 7. CV_ASSUME_DISTID 설정으로 os 버전 8.4로 설정 (지원 안 되는 버전만 실행)

```
[root@psk1 ~]# cat /etc/os-release
NAME="Oracle Linux Server"
VERSION="8.9"
ID="ol"
ID_LIKE="fedora"
VARIANT="Server"
VARIANT_ID="server"
VERSION_ID="8.9"
PLATFORM_ID="platform:el8"
PRETTY_NAME="Oracle Linux Server 8.9"
ANSI_COLOR="0;31"
CPE_NAME="cpe:/o:oracle:linux:8:9:server"
HOME_URL="https://linux.oracle.com/"
BUG_REPORT_URL="https://github.com/oracle/oracle-linux"

ORACLE_BUGZILLA_PRODUCT="Oracle Linux 8"
ORACLE_BUGZILLA_PRODUCT_VERSION=8.9
ORACLE_SUPPORT_PRODUCT="Oracle Linux"
ORACLE_SUPPORT_PRODUCT_VERSION=8.9
```

``` 각 노드 .bash_profile에 추가
[oracle@psk1 ~]$ vi .bash_profile

# 아래 두 줄 추가
export LANG=en_US.UTF-8
export CV_ASSUME_DISTID=OEL8.4

[oracle@psk1 ~]$ source .bash_profile
[oracle@psk1 ~]$ echo $CV_ASSUME_DISTID
OEL8.4

[oracle@psk2 ~]$ vi .bash_profile

# 아래 두 줄 추가
export LANG=en_US.UTF-8
export CV_ASSUME_DISTID=OEL8.4

[oracle@psk2 ~]$ source .bash_profile
[oracle@psk2 ~]$ echo $CV_ASSUME_DISTID
OEL8.4
```
