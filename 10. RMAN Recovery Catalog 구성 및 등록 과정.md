## 요약

### 1. 원격지 카탈로그 서버(Linux7) 네트워크 환경 구축

네트워크 인터페이스 수정: enp0s3를 고정 IP(10.0.2.76)로 설정하여 RAC 노드와의 통신 경로 확보

리스너 설정 및 기동: listener.ora 파일의 HOST 정보를 수정하고 lsnrctl start를 통해 1521 포트를 오픈

방화벽 해제: systemctl stop firewalld를 통해 RAC 노드로부터의 인바운드 트래픽 허용

### 2. 카탈로그 전용 저장 공간 및 유저 생성

데이터베이스 기동: startup을 통해 orcl 인스턴스 활성화

테이블스페이스 생성: 카탈로그 메타데이터가 저장될 전용 공간(rc_ts) 확보

전용 유저 생성 및 권한 부여: rc_user 생성 후 recovery_catalog_owner 롤을 부여하여 관리 권한 할당

### 3. RMAN Recovery Catalog 생성 및 RAC 등록

카탈로그 스키마 생성: rman catalog 접속 후 CREATE CATALOG 명령으로 메타데이터 관리용 베이스 테이블 생성

TNS 정보 등록: RAC 노드의 tnsnames.ora에 원격지 카탈로그 서버 정보 추가 및 tnsping 확인

Target DB 등록: rman target / catalog 접속 후 REGISTER DATABASE를 실행하여 RAC 노드의 DBID와 백업 정보를 카탈로그 서버와 동기화
